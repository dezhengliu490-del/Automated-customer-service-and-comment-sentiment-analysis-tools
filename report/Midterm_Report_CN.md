# 期中项目报告

## 项目标题 (Project Title)

自动化客服与评论情感分析系统

## 项目目标 (Project Goal)

本项目的首要目标是构建一个智能平台，大幅降低商家的客服成本，并主动从消费者的原始评论中精准提取产品缺陷与痛点。

**对 AI、IT、工程和商业领域跨学科合作的促进作用：**
本项目深度融合了以下多个学科领域的知识与技术：

* **人工智能 (AI)：** 借助最前沿的大语言模型 (LLM)，结合检索增强生成 (RAG) 与提示词工程（少样本提示、思维链），实现深度的语义理解、情感分类以及自然且拟人化的文本生成。
* **IT 与软件工程：** 项目贯彻了严谨的工程实践，包括运用 Streamlit 构建响应式前端界面、开发基于异步机制的高并发后端 API 请求逻辑、部署和调优本地向量数据库，以保障端到端数据链路的高效与稳健。
* **商业管理 (Business)：** 核心技术的驱动力源于其实际的商业落地价值。系统不仅实现了拟人化交互，还将模糊的客户情绪量化，进而总结出具体的产品痛点（例如“包装破损”等），为商铺的供应链打磨和产品迭代提供真实可靠的数据驱动决策支持，最终达到降低退换货损失和提升整体销量的商业目的。

## 商业构想 (Business Idea)

中小电商和本地生活商家往往受限于人力成本，难以及时回复海量的客户咨询，同时也缺乏充足的资源来对成百上千条买家评价进行系统性的分析复盘。

我们的商业构想是通过此项目提供一套极具可用性的“AI 驱动型 SaaS 解决方案”：
它既是一个能根据商家专属“退换货规则知识库”和“语气词设定”自动进行高度拟人化接待的**智能客服助理**；
同时，它也是一个能将非结构化文本情绪瞬间转化为清晰的可视化商业报表的**数据分析引擎**，帮助商家及时察觉异常差评趋势、持续优化产品体验并阻断售后亏损。

## 系统设计 (System Design)

**硬件设施 (Hardware)：**

* 用于本地开发、脚本编写测试以及部署服务的标准 PC 设备/服务器。
* 用于支持大模型高速推理计算的云端 API 基础设施。

**软件及技术栈 (Software)：**

* **前端交互：** Python + Streamlit 框架（用于零前端基础构建专业交互网页与可视图表），配合 Altair 和 Pandas 进行页面级数据处理和图表渲染。
* **后端开发：** Python，结合 `asyncio` 与 `httpx` 处理 LLM 接口的高并发异步请求。
* **AI 核心层：** 接入 Google Gemini / DeepSeek 等顶尖大模型 API 服务。
* **数据与 RAG 架构：** 采用 LangChain / LlamaIndex 框架搭建轻量级本地向量数据库（如 ChromaDB/FAISS），用于实现商家文本的分块存储（Chunking）与语义检索（Embedding检索）。

## 后续计划 (Future Plan - After Midterm)

在成功跑通最小可行性产品 (MVP) 链路并顺利通过期中里程碑考核后，我们在第 9 至 16 周的后续计划包括：

* **测试与架构升级 (Testing & Architecture)：** 我们将引入专门的“边界极端数据集”（如正话反说、乱码、全表情包评价），并实装“防御性提示词 (Defensive Prompt)”，彻底遏制大模型产生幻觉。我们即将引入 LangChain 等框架正式搭建轻量级的本地 RAG 向量知识库，同时针对接入该 RAG 知识库的情景，给出量化的准确率 A/B 测试报告。
* **系统改进 (Improvement)：** 执行全面的后端代码重构与规范化，优化 Token 桶等并发限流机制，解决处理大文件时的性能瓶颈；并且大幅提升用户体验，比如增加“一键将图表与 AI 结论导出为 CSV 或快照报告”的功能。
* **最终展示计划 (Final Demonstration Plan)：** 我们计划在期末严格落实项目特征冻结（Feature Freeze），开展毫无死角的端到端联调测试；撰写基于真实数据的《商业洞察分析报告》；录制一套 3-5 分钟的备用兜底实机演示视频；团队成员将多次进行带有计时和“红蓝对抗防守”策略的实战模拟演讲，确保以最高标准通过期末终期答辩。

## 当前进度百分比 (Progress So Far)

**整体项目进度：50%**
我们已按甘特图要求顺利完成第一阶段和第二阶段的任务。期间攻克了 API 频繁限流的网络难题，构建了 200 条规模的黄金验证数据集（Ground Truth），成功迭代出 V2.0 版的提示词框架（将情感分类精度拉升至 80% 以上），通过提示词硬注入（Prompt-Stuffing）的方式初步实现了商家语气的克隆，并在前端实现了基本的可视化与聊天框架。系统当前正准备全面过渡到标准的 RAG 向量检索架构开发阶段。
